# 宝盈基金互联网风向看板 - 快速上手指南（GitHub线上版）

更新时间：2026-04-13
适用对象：首次接触本项目的运营、投放、内容、管理同事。

## 1. 一句话说明
这套看板用于追踪互联网平台基金“加仓/减仓风向”、基金详情页用户行为与竞品弱点，支持日常盯盘、复盘、选题和策略调整。

## 3. 每张看板看什么
### 3.1 核心看板
用途：快速看加仓榜/减仓榜/双边表/当日排序。
重点：
- 近7日、近2周、近1月上榜次数
- 近1周、近2周、近1月、日涨跌幅
- 基金代码为文本格式（保留前导 0）
建议：日常先看这里，再跳转到其他看板。

### 3.2 动态指标看板
用途：把运营常用指标做成“可执行指标清单”。
重点：
- 风向核心：热度、净偏好、趋势延续
- 收益验证：热度是否有收益支撑
- 信号标签：短期热点/趋势延续/分歧预警/退潮观察
建议：用于日报、周会、策略说明。

### 3.3 竞品弱点看板
用途：识别可突破的同业产品。
重点：
- 综合弱点分
- 热度空转度、分歧度、衰减度、名次脆弱度、连续性断层率
建议：按弱点分排序，挑“高优先狙击”标的制定动作。

### 3.4 基金详情驾驶舱
用途：从基金详情页看用户行为与内容热度。
重点：
- 浏览、关注、持有转化（曝关比、关转比等）
- 动态笔记互动趋势
- 跟踪误差分层（第一梯队/第二梯队/偏弱）
建议：用于内容方向、转化链路和产品叙事优化。

### 3.5 指标文档
用途：指标词典。
内容：每个指标的定义、计算方式、目的、运营动作建议。
注意：若页面显示“指标文档暂未生成”，说明本次自动更新未产出文档文件，其他看板不受影响。

## 4. 数据更新时间规则
- 定时任务：工作日 16:00（Asia/Shanghai）自动执行更新+部署。
- 核心看板内“手动更新按钮”在 GitHub Pages 静态站点不可用（已禁用文案提示）。
- 若当天 16:00 前查看，可能看到“更新至前一日”属于正常。

## 5. 新人标准使用流程（建议）
1. 先看核心看板：确认今天热点基金与榜单方向。
2. 跳动态指标：判断是短期噪音还是趋势延续。
3. 跳竞品弱点：找同业可突破点。
4. 跳驾驶舱：验证用户行为和内容互动是否匹配。
5. 需要汇报时，再到指标文档复制口径说明。

## 6. 常见问题
### Q1：为什么部署成功了但页面看起来没变化？
A：浏览器缓存。请强刷（macOS：`Cmd + Shift + R`）。

### Q2：为什么有时部署失败？
A：常见原因是 Pages 设置、Secrets 配置或外部接口波动。当前工作流已加入预检与自动重试。

### Q3：我只做运营，不懂代码，如何确认是否更新完成？
A：看 GitHub Actions 最新 run 是否为绿色 `success`，再访问线上地址确认页面时间与数据日期。

## 7. 维护人需要知道的最小配置
仓库 Secrets（Actions）需要：
- `CD_NAMESPACE_NAME`
- `CD_USER_NAME`
- `CD_PASSWORD`
- 可选：`CD_TARGET`、`CD_BROWSER_ID`
- 可选增强：`GH_PAGES_ADMIN_TOKEN`（用于自动修复 Pages 启用状态）
